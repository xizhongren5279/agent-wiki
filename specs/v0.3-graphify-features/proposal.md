# Proposal: Agent-Wiki v0.3 — 借鉴 Graphify 的 8 项增强

## 背景

Agent-Wiki v0.2 是一个 MVP 知识库插件，具备摄入、标签、编译、搜索、文章生成、健康检查 8 个核心工具。但在实际使用中暴露了几个短板：

1. **搜索太浅** — 纯标签 + 关键词匹配，无法理解语义关联
2. **知识孤岛** — 页面之间没有显式关系，无法发现跨主题联系
3. **增量不完整** — 同名文件内容变了不会重新处理
4. **来源不透明** — 无法区分"原文摘录"和"LLM 推断"
5. **无法摄入网页** — 只支持本地文件

研究了 Graphify（知识图谱提取系统）后，筛选出 8 个在"零外部依赖"约束下可实现的功能。

## 目标

在保持 agent-wiki 核心设计原则（零 LLM、零外部依赖、纯文件 I/O）的前提下，增加 8 项功能：

| # | 功能 | 解决什么问题 |
|---|------|-------------|
| 1 | 内容变更检测（SHA256） | 同名文件内容变了能自动重新处理 |
| 2 | 置信度/来源标记 | 知道每条知识是摘录、推断还是问答衍生 |
| 3 | 关系提取 + 双向链接 | 页面之间建立显式关联 |
| 4 | 基于关系的搜索增强 | 搜索结果自动扩展关联页面 |
| 5 | 主题自动聚类 | 共享标签的页面自动归组 |
| 6 | 知识图谱报告 | 一页摘要鸟瞰知识库健康度 |
| 7 | URL 摄入 | 支持从网页抓取内容 |
| 8 | 去重检测 | 摄入时发现重复内容 |

## 约束

- 不引入任何新 npm 依赖
- 不需要 embedding 服务、图数据库、tree-sitter 等外部组件
- 所有智能由宿主 Agent 的 LLM 提供
- 保持 MCP stdio 传输协议不变

## 验收标准

- 所有 8 个功能可通过 MCP 工具调用
- 现有 8 个工具的接口向后兼容
- state.json 结构向后兼容（新增字段，不删旧字段）
- SCHEMA.md 自动更新新功能说明
